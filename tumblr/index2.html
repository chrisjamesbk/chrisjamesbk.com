    Creating a repository in GitHub

    I’ve been using git for nearly a year now, ever since I started at OpenSky. Although I’m a long way away from proclaiming myself a git master, I have become comfortable in performing most everyday git actions and even some more advanced actions. I’m comfortable forking repositories, creating submodules, performing interactive rebases and cherry-picking branches.

    Despite all of this, I’d never created my own GitHub repository until today. Despite soon realizing that git is a far superior version control system, I’ve carried on using svn for my personal and freelance projects. Mostly this is because I am comfortable with my workflow, and my web host offers an easy way to set up self-hosted svn repositories. However, I am looking to use git more, and today was a good first step on that path.

    It turns out that it’s very simple to create a repository in GitHub. Here’s how:

    1. Click on the ‘New Repository’ button

    2. Enter repository details

    • Name (required)
    • Description (optional)
    • choose Public (free) or Private (requires paid account)
    • select ‘Initialize this repository with a README’ (recommended if this is a new project)
    • Click ‘Create Repository’ button

    3. If you selected ‘Initialize with a README’, you can now clone the repository to your local dev environment

    // in unix terminal
            // navigate to the directory where you want the repo directory to go
            $ cd path/to/local/repo/location 
    // get the repository SSH path
            // you can find this by navigating to your repo, and selecting it here
            

    // clone the repo. get the uri from GitHub
            // this adds the repo 'repo-name' in the directory you are in
            $ git clone git@github.com:username/repo-name.git

    Note: if you clone the http version the the repo, you will be prompted for your GitHub username/password every time you try to commit changes. You can fix this by replacing the http path with the ssh path in .git/config.

    EDIT

    4. It is also good practice too add a .gitignore file to the root of the repository. .gitignore is a hidden file that tells git which files and directories to ignore (e.g. don’t add/commit/push/etc) in your cloned repository. An example of the type of file you might want to ignore are the .DS_Store files that OSX adds to each directory. If you fail do this step with a clean repo, then this will help you sort it out.
